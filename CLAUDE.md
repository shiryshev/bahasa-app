# Lala Bahasa — гид для Claude Code

PWA-тренажёр индонезийских слов (bahasa Indonesia) для одного пользователя, живущего на Бали. Открывается в браузере/на телефоне, работает офлайн, ставится иконкой. Ванильный JS, без сборки.

## Структура репозитория

- `index.html` — весь UI и логика в одном файле.
- `base.js` — данные словаря + полный бэкап. **Генерируется**, руками не править.
- `sw.js` — service worker (офлайн, установка иконки). Содержит версию кэша `CACHE`.
- `manifest.webmanifest`, `icons/` — PWA-обвязка.
- `build_base.py` — скрипт сборки `base.js` из выгрузки Notion.
- `HANDOFF.md`, `database-contract.md` — контекст проекта и правила базы слов.

## Деплой

- Хостинг: GitHub Pages, репозиторий `bahasa-app`, адрес `https://shiryshev.github.io/bahasa-app/`.
- Источник Pages — ветка `main`, папка `/(root)`.
- **Деплой = коммит и пуш в `main`.** GitHub Pages подхватывает автоматически.
- После правки `index.html` — **обязательно** поднять версию `CACHE` в `sw.js` (`v12` → `v13` …), иначе телефон не подхватит обновление. `base.js` тянется network-first, версию бампать не обязательно.

## База слов — правила (жёсткие)

- Источник истины один — страница Notion «📒 Bahasa Dictionary». Только она правится вручную.
- `base.js` — производное от Notion. **Не редактировать вручную, только генерировать** через `build_base.py`.
- У каждой записи постоянный числовой `id`. Никогда не меняется и не переиспользуется, даже после удаления. Скрипт хранит `LALA_NEXT_ID` (только растёт).
- Пополнение — только в Notion, строкой `- **indonesian** · pronunciation · russian` (без `#id`, его проставит сборка).
- Формат записи: `#id · **indonesian** · pronunciation · russian`.
- Удаление слов инициирует приложение (кнопка «🗑 Удалить из словаря» → список в экране «Прогресс»); строки удаляются из Notion, затем пересборка.
- Подробности и роли — в `database-contract.md`.

## Сборка базы (Notion → приложение)

1. Скопировать строки из раздела `## Записи` страницы Notion в `notion_dump.txt`.
2. `python3 build_base.py notion_dump.txt` → обновится `base.js`, создастся `notion_normalized.txt`.
3. Если появились новые `#id` (кто-то добавил слова без номера) — залить `notion_normalized.txt` обратно в Notion, чтобы номера зафиксировались.
4. Коммит + пуш `base.js` в `main`.

## Как вести работу

- Правки приложения — в `index.html` (+ бамп `CACHE` в `sw.js`).
- Перед коммитом — прогонять JS через `node --check` и, где можно, юнит-тесты логики.
- Не газлайтить пользователя: при расхождении сначала проверять свою версию (кэш, недокачанный деплой, свой баг), потом уже гипотезу об ошибке пользователя.
- Стиль общения: на «вы», кратко и по делу.

## Данные приложения (localStorage)

`lala_progress_v1`, `lala_settings_v1`, `lala_session_v2`, `lala_excluded_v1`, `lala_daily_v1`.

## При переезде на Claude Code — доделать

- Добавить в репозиторий `HANDOFF.md`, `database-contract.md`, `build_base.py` (сейчас лежат только в папке проекта Claude, в git их нет).
- В git-репозитории файлы приложения лежат в **корне** (`index.html` в корне), а `build_base.py` писал в `lalabahasa-app/base.js`. Поправить путь вывода скрипта на корневой `base.js`.
- Для сборки базы из Notion — подключить Notion как MCP-коннектор в Claude Code (иначе шаги 1 и 3 сборки делаются вручную).
