# Lala Bahasa — передача контекста новому чату

Прочитай этот файл и `database-contract.md` в начале работы — в них всё состояние проекта. Историю переписки новый чат не видит, но всё важное лежит в файлах и в Notion.

## Что это

`Lala Bahasa` — PWA-тренажёр индонезийских слов (bahasa Indonesia) для одного пользователя, живущего на Бали. Открывается в браузере/на телефоне, работает офлайн, ставится иконкой на экран.

## Где что лежит

- `lalabahasa-app/` — само приложение:
  - `index.html` — весь UI и логика в одном файле (ванильный JS, без сборки).
  - `base.js` — данные словаря для приложения + полный бэкап. **Генерируется**, руками не править.
  - `sw.js` — service worker (офлайн, установка иконки). Есть версия кэша `CACHE` (сейчас `lala-bahasa-v11`).
  - `manifest.webmanifest`, `icons/` — PWA-обвязка.
- `build_base.py` — скрипт сборки `base.js` из Notion.
- `database-contract.md` — **правила базы слов** (источник, формат, ID, пополнение, удаление, сборка). Обязательно к прочтению.
- `Bug Report/` — скриншоты багов от пользователя.

## Хостинг и деплой

- GitHub Pages, репозиторий `bahasa-app` (владелец — пользователь), адрес `https://shiryshev.github.io/bahasa-app/`.
- **Деплой делает пользователь**: заливает изменённые файлы в репозиторий вручную. У ассистента нет прав писать в GitHub.
- После правки `index.html` — поднимай версию `CACHE` в `sw.js` (v11 → v12 …), иначе телефон не подхватит обновление. `base.js` тянется network-first, версию бампать не обязательно.

## База слов

- Источник истины — страница Notion «📒 Bahasa Dictionary» (`https://app.notion.com/p/38cec2754c7e81188f86dcd80ada53a2`). Сейчас **114 записей**, ID 0–116 (есть пропуски от удалений), `LALA_NEXT_ID = 117`.
- Схема записи: `#id · **indonesian** · pronunciation · russian`. Полей `type`, `note`, `дата` — нет.
- Менять базу — только через Notion, потом `python3 build_base.py notion_dump.txt`. Подробности и роли — в `database-contract.md`.

## Что уже умеет приложение

- 4 упражнения: написание рус→бахаса, написание бахаса→рус, на слух (бахаса→рус, озвучка TTS `id-ID`), произношение (рус→бахаса, распознавание речи + кнопки «Знаю» / «Не помню»). Плюс режим «Все упражнения».
- Сессии по 10/20/30 слов; интервальные повторения (Leitner, `MASTER_BOX = 4`); прогресс по словам и упражнениям, % верных; экран «Прогресс» с оценкой уровня; список выученных слов с кнопкой «↩ на обучение»; удаление слова из словаря (обратная связь для чистки базы).
- Умный матчинг русских ответов: числа = слова, регистр, порядок слов, необязательные слова, слитные формы, опечатки.
- Данные в браузере: `localStorage` — `lala_progress_v1`, `lala_settings_v1`, `lala_session_v2`, `lala_excluded_v1`.

## Как правильно вести работу

- Правки приложения — в `lalabahasa-app/index.html` (+ бамп `CACHE` в `sw.js`). После — попросить пользователя залить в GitHub.
- Перед сдачей — прогонять JS через `node --check` и, где можно, юнит-тесты логики (в этом проекте так и делали).
- Не газлайтить: при расхождении сначала проверять свою версию (кэш, недокачанный деплой, свой баг), потом уже гипотезу об ошибке пользователя.
- Стиль общения с пользователем: на «вы», кратко и по делу, без воды.
